# Codex 一発出しプロンプト（ユニクロデザインのスライドLP）

下の「===PROMPT===」以降を**まるごとコピーして Codex に貼る**だけ。Codex がリポジトリ内の設計図を読み、1枚のHTMLスライドLPを生成します。

===PROMPT===

あなたは一流のプレゼンデザイナー兼フロントエンドエンジニアです。このリポジトリ内のスライド設計図から、**ユニクロ（UNIQLO）のブランド世界観**の、単一HTMLファイルのスライドLPを作ってください。

## 入力（必ず読む）
`talk-materials/kanto-ai-education-2026.md` を開き、**「# 本編スライド（全21枚）」から「## S備」まで**を使う。各「## Sn ｜ タイトル」を1スライドにする。「> 話者ノート」はスライド面に出さず、presenter notes として隠す。〔要・実データ〕〔要確認〕〔※…〕のプレースホルダは消さずそのまま残す（あとで差し替えるため、赤い破線の枠で軽く目立たせる）。設計図の上部（設計ログ）と下部（付録・想定問答）はスライドにしない。

## 出力
- `talk-materials/slides-uniqlo.html` を**1ファイル完結**で生成（ビルド不要・ダブルクリックで開ける）。
- 外部依存は Google Fonts（Noto Sans JP）のみ可。JS/CSSはインラインで自己完結。

## デザイン＝UNIQLO
ユニクロの店舗・サイト・LifeWear広告の視覚言語を厳密に再現する。

- **配色**：背景は純白 `#FFFFFF`。文字は黒 `#111111`。アクセントは **UNIQLO RED `#FF0000` ただ1色**。補助に薄いグレー `#F5F5F5`。グラデーション・影・角丸・絵文字は使わない（フラット＆シャープ＆直角）。
- **タイポグラフィ**：和文 `Noto Sans JP`（見出しは900、本文は500/700）。欧文・数字は `Helvetica Neue, Arial, sans-serif`。見出しは**大きく・詰めて・太く**。英語ラベルは大文字。
- **グリッド**：厳格なモジュラーグリッドと**広い余白**。情報を詰め込まない。1スライド＝1メッセージ。
- **ユニクロの象徴的モチーフを使う**：
  - スライド番号や章ラベルを **赤い正方形タグ**（赤地に白文字、ロゴのタグ風）で表示。
  - 和文の隣に**小さな英語キャプション**を添える（例：課題＝ISSUE、これからの価値＝WHAT'S NEXT）。
  - キーメッセージの一語だけ**赤の塗りハイライト or 赤の下線**。
- **タイトル(S1)・スローガン(S6)・締め(S21)** は、**全面赤 `#FF0000` 背景＋白の極太大文字**でフルブリードに（反転して最も印象的に）。
- 通常スライドは白地。左上に赤タグでスライド番号、上部に大きな黒の見出し、その下にキーメッセージ（やや大きめ）、本文は短い箇条書き。右上隅に〔初級/中級/上級/全員〕の小さな赤アウトラインのチップ。
- **表**（S9・S18）は罫線最小・余白広めのクリーンな表。Before/After(S18)は左右2カラムで「以前→今」を対比、"今"側を赤で強調。
- **S5**（作る側の負担）は「テーマ→文献→スライド→進行→振り返り」を**横一列の矢印フロー**に。
- **S11**（作るAI／見張るAI）は**左右2カラムの対比**レイアウトに。

## 技術仕様
- 各スライドは**フルビューポート(100vw×100vh)・16:9基準**のセクション。`scroll-snap` で**LPとしてスクロール**もでき、**キーボード（← → / Space / ↑ ↓）でも送れる**ようにする。
- 右下に小さな**ページ番号（n / 21）**。
- `N` キーで presenter notes をトグル表示（既定は非表示）。
- `F` キーでフルスクリーン。
- 文字は投影前提で大きめ。スマホ幅でも崩れないようフォントを clamp で可変に。
- 日本語の禁則・行間に配慮（line-height 1.5前後、見出しは詰める）。

## 厳守
- 設計図の**文言を改変しない**（誤字修正以外）。内容を勝手に足さない・要約しすぎない。
- スライド面に専門用語（ハルシネーション／LLM／監視エージェント等）を出さない（設計図側で言い換え済み）。
- 完成後、生成したスライド枚数（21＋備え）と、差し替えが必要なプレースホルダの一覧を最後に報告する。
